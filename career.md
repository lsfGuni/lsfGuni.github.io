# Career

안녕하세요.  
저는 **자동화와 근본 원인 해결로 안정적인 인프라를 만드는 엔지니어** 한권희입니다.

백엔드 개발자로 커리어를 시작했지만,  
실무에서는 점차 **Linux 서버 운영, AWS와 온프레미스 연동, CI/CD 자동화, 모니터링, 보안 대응, 장애 분석, 운영 표준화**까지 역할을 확장해 왔습니다.

저는 기능 개발 자체보다,  
**서비스가 실제 환경에서 안정적으로 동작하도록 만드는 구조와 운영 방식**에 더 큰 관심을 가지고 있습니다.

문제가 발생했을 때는 단순 복구에 그치지 않고,  
**원인 분석 → 해결 → 고도화 → 문서화**까지 이어지는 방식으로  
재발을 줄이고 재현 가능한 운영 체계를 만드는 것을 중요하게 생각합니다.

## Work Style

- 문제를 단편적으로 보지 않고 **서비스 흐름 전체**를 따라가며 원인을 좁혀갑니다.
- 장애 대응 시 **임시 복구보다 재발 방지**를 더 중요하게 생각합니다.
- 반복 작업은 자동화하고, 운영 절차는 문서화하여 **재현 가능한 운영 환경**을 만듭니다.
- 개발, 운영, 네트워크를 분리하지 않고 **하나의 시스템**으로 이해하려고 합니다.
- 팀 내 역할을 존중하면서도, 필요한 경우 직접 구조를 정리하고 실행하는 편입니다.
